# 用小组 PPT 理解 Git

## 传统协作的问题
在传统的艺术或学术项目中，小组合作制作幻灯片往往会陷入“命名学灾难”。
举例：我们经常会看到 `汇报PPT_A改.pptx`、`汇报PPT_B改.pptx`、`合并版.pptx`、`最终版.pptx`。这种依靠文件名来区分历史的方法，不仅容易导致版本覆写，更无法追踪“谁在何时为何做出了修改”。

## Git 怎么解决
Git 引入了平行的叙事结构。各自在一个平行的分支上工作。完成草图后，通过发起合并请求（Pull Request）让组员审查（Review），确认无误再汇入主干（Merge）。

## 注意事项
Git 是一种基于文本差异的比对工具，不适合多人同时编辑同一个二进制的 .pptx 文件。它更适合对项目大纲、核心文案进行版本控制。
